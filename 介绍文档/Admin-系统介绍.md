# 山药铭记后台管理系统 - 介绍

## 1. 系统概述

本系统是“山药铭记”中医学习微信小程序的总控后台及资源整编端。采用目前主流的 Vue 3 + Vite + Node.js 的前后端分离架构体系。除了承载常规的小程序用户登录确权与系统运营看板以外，核心使命是对系统中包含的诸多重度垂直中医药学类数据库提供精细化的修改及统筹支撑。

## 2. 架构技术栈

### 前端技术栈 (Admin 端)
- **底层驱动模式**: Vue 3 (全方位普及 `<script setup>` 组合式 API 开发模型)
- **打包与构建体系**: Vite 5.x 极速联调支撑
- **视图层路由分配**: Vue Router 4.x
- **状态树管理**: Pinia (轻巧而现代的状态池管理机制)
- **UI 呈现组件库**: 搭配 Element Plus，以及 vxe-table 带来的高性能大数据表格展示解决方案
- **开发及静态检查**: 强依赖 TypeScript 进行面向对象与安全的数据类型校验，辅以 ESLint 控制
- **样式处理器**: Scss (Sass) 进行底层变量级把控，以及 UnoCSS 作为原子化按需样式渲染增强
- **网络层封装代理**: Axios

### 后端技术栈 (Backend 端)
- **运行环境**: Node.js
- **服务端主框架**: Express.js
- **数据库支撑及 ORM 映射**: 选用 MySQL 关系型数据库，配合 Sequelize JS 对象关系映射操作模型保障查询安全性。
- **权限边界与安全控制**: JWT 全局鉴权拦截校验，同时启用 CORS 中间件防范以及预检拦截。

## 3. 基于中医业务的核心管理模块

目前基于项目功能设计，其对应视图分布在了 `admin/src/pages/admin/` 目录结构下：

### 3.1 核心医库与教材内容体系 (Content & Categories)
这是整个山药铭记生态的基础资料枢纽。用以管控诸如《伤寒论》、《金匮要略》、中药药方、针灸经络等巨量古典条文以及知识点的编辑修订后台。
- **条文分类管控(Categories)**：根据不同朝代以及医学科目构建灵活可扩展的内容目录树形类别。
- **内容字典增删查改(Content)**：处理文本级的勘误（对古籍常见错漏、避讳字的专项校对与数据结构修补维护）。

### 3.2 社区与学风运营体系 (Feedback)
- **答疑与问题追踪**：追踪、受理小程序端用户的建议。主要是针对古文录入可能存在错误的提报追踪，或者中医学习互助区用户讨论帖子的审计运营。

### 3.3 积分布局与扩展资料站 (Shop & Materials)
- **积分商城生态 (Shop)**：上架或下架可以通过学习进度或者邀请新用户获取来的积分，并在商城内允许兑换的小型测试礼品或数字权益。
- **资料专区控制 (Materials)**：管理后端存放的高规格进阶扩展讲义、精校PDF原版书等医学辅件的发放与阅览权限。

### 3.4 用户体系数据与服务预警
- **统一用户池 (Users)**：提供封停异常作弊账号及查询微信侧回流用户整体积分画像功能保障。
- **滚动跑马灯公告 (Notices)**：管理员可通过此功能给所有用户小程序端的首页下发全量即时滚动文字广播提示。
- **数据走势及宏观洞察看板 (Stats)**：收集小程序的实时打卡、背诵通关分布率及核心表单流量的数据支撑工具。

## 4. 后端路由网关结构全览
在 `backend/routes/` 下设立的标准 RESTful 模型分发控制器：
- `/routes/auth.js` 与 `/routes/user.js` - 处理通过前端拿到的 `wx.login` Code 交换确立用户的 session / JWT 认证会话操作及基本信息迭代。
- `/routes/content.js` - 重度使用，负责提取分发和投递不同中医细分条线类别条目到页面的查询引擎。
- `/routes/progress.js` - 数据落盘分析链路核心接口：对用户的学习进境（背了多少、看了多少）与核心记忆算法反馈数据进行持久化更新落表操作。
- `/routes/material.js` 与 `/routes/shop.js` - 提供获取与消耗（兑换处理的原子化级联处理）商品 / 辅导材料的一致性接口。

## 5. 项目部署备忘 (Deployment Workflow)
Admin项目全面切换至高效的 `pnpm` 包管理工具，本地编译可采用以下命令规程：
```bash
# 进入后台目录并同步最新依赖版本库
cd admin
pnpm install

# 唤醒热重载开发服务器监听模式 (Default to localhost:5173 / 其他动态分配口)
pnpm dev

# 在生产部署前，由 vite 及 tsc (vue-tsc) 执行压缩及格式校验打包工序
pnpm build
```
对于后端 Node 的运行部署工作，则推荐管理员自行配置 PM2 守护进程体系搭配合理有效的 `.env` 文件处理敏感的环境切分设定（如保护 Database 凭据与 WeChat Secret Keys 层面的解耦）。
